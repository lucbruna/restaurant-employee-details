# Bhukkad Requirements Bundle

This folder collects a clean, shareable markdown pack for explaining Bhukkad to stakeholders, contributors, and agentic AI systems.

## Files

- `linkedin-post.md`
  Long-form LinkedIn post draft explaining what Bhukkad is, why it matters, how it is built, and why its free-to-use positioning is important.

- `project-overview.md`
  Concise product summary, problem statement, target users, differentiators, and key modules.

- `tech-used.md`
  Practical breakdown of the technology stack across frontend, backend, data, auth, realtime, validation, and developer tooling.

- `architecture.md`
  Runtime and system architecture explanation covering request flow, auth boundaries, order orchestration, realtime contracts, and operational design decisions.

- `agentic-ai-handoff.md`
  A handoff document intended for coding agents that need to understand how to work inside the Bhukkad codebase without breaking its product assumptions.

- `tech-mindmap.md`
  Mermaid mindmap showing the Bhukkad technology stack and how the pieces group together.

- `architecture-mindmap.md`
  Mermaid mindmap showing the major runtime layers and flows inside Bhukkad.

## Notes

- The folder name uses the exact spelling requested: `REQIREMENTS`.
- The content is grounded in the current Bhukkad repository, especially the existing `README.md`, `docs/ARCHITECTURE.md`, `docs/TECH_STACK.md`, `docs/DATABASE_SCHEMA.md`, `docs/DEVELOPMENT_WORKFLOW.md`, `docs/PAGE_BLUEPRINTS.md`, `docs/UI_SYSTEM.md`, `docs/RECREATION_GUIDE.md`, `docs/bhukkad-copy.md`, and key runtime files such as `server.ts`, `middleware.ts`, `lib/auth.ts`, and `lib/orders/create-order.ts`.
